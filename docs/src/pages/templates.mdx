# Templates [WIP]

Templates is a new feature that uses theme options and a pre-built set of layout components to create powerful layouts for you.

## Usage

All you have to do is import the `ElementsTemplate`, supply a few custom components as props, and wrap it around your application's page content.

Here's an example of a root layout:

```jsx
import React from "react"
import { ElementsTemplate } from "gatsby-theme-elements"

// Custom components

import FooterWidgets from "../components/FooterWidgets"
import MobileNav from "../components/MobileNav"
import Topbar from "../components/Topbar"
import Footer from "../components/Footer"

// Nav menu links

const menuItems = [
  { label: "Home", path: "/" },
  { label: "About", path: "/about" },
  { label: "News", path: "/news" },
  { label: "Contact", path: "/contact" },
]

export default ({ children }) => (
  <ElementsTemplate
    menuItems={menuItems}
    topbar={<Topbar />}
    mobileNav={<MobileNav />}
    footerWidgets={<FooterWidgets />}
    footer={<Footer />}>
    {children}
  </ElementsTemplate>
)
```

`NOTE:` You don't need to add semantic markup to your components.

Elements takes care of this, so in most cases you can wrap your components in React Fragments.

## When to Use: Components vs. Templates

Templates makes the process of creating a layout easier and cleaner, but it inhibits your ability to add detail or complex customizations.

If you need to style or access individual components of a layout, use [Layout Components](/layout-components).

If you only need to customize a few things like colors or fonts, try using a template. Your configurations in `options.js` and `theme.js` handle most of the heavy lifting.

The `ElementsTemplate` lets you supply custom components for:

- Topbar
- Logo
- Navigation menu items
- ColorToggle
- MenuToggle
- MobileNav
- HeaderWidgets
- Sidebar
- SideNav
- FooterWidgets
- Footer
- TabBar

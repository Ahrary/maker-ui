## Semantic Element

`<aside />`

| Props               |            Type             |                                                                                                                                        Description |
| ------------------- | :-------------------------: | -------------------------------------------------------------------------------------------------------------------------------------------------: |
| **children**        |           _node_            |                                                                                                                                           required |
| **backgroundColor** |          _string_           |                                                                                        overrides the default `bg_topbar` color value in `theme.js` |
| **border**          |          _string_           | shorthand border syntax that overrides the default `1px solid` border bottom and border color designated in `theme.js`. Example: `"1px solid red"` |
| **color**           |          _string_           |                                                                                          overrides the `text_topbar` color value set in `theme.js` |
| **maxWidth**        | _string_, _number_, _array_ |                                 overrides the default `topbar.maxWidth` value in `options.js`. Can take an array of values for responsive scaling. |
| **sticky**          |          _boolean_          |                                                                                        overrides the default `topbar.sticky` value in `options.js` |

## Styling

You can add extra styles to the header with the `sx`, `css`, or `style` prop (ThemeUI, Emotion, or React).

`NOTE:` these styles will be aplied to the Topbar's nested container. To access any of the root styles, use the props listed above.
